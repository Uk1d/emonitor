# eTracee 文档汇总

## 概览
- 项目目标：基于 eBPF 的轻量级安全可观测与攻击链可视化。
- 运行环境：Linux 采集、Go 用户态服务、Web 前端（内置静态页面）。
- 当前默认行为：HTTP/WS 绑定 `0.0.0.0:8888`，无需鉴权；可通过环境变量调整。

## 架构要点
- eBPF 层（`src/bpf`）：事件结构 `etracee.h`，主程序 `etracee_main.c`，子探针 `execve_trace.c`、`filesystem_trace.c`、`network_trace.c`、`security_trace.c`。
- Go 用户态（`src/go`）：事件消费与转换、规则引擎、告警与图谱、Web API。
- Web 服务与静态资源：`internal/api/http/router.go` 路由注册，`alert_api.go` WS/REST，`internal/web/static/index.html` 前端页面（D3 可选）。
- 配置：`config/` 下安全规则与存储配置；环境变量与运行参数在 `internal/common/config/config.go` 与 `main.go`。

## 事件采集与推送
- eBPF 通过 tracepoint 捕获事件，RingBuffer 传递到用户态。
- 用户态解析为 `EventJSON`，应用过滤与规则，再广播到 WS 客户端；并按 5 秒推送统计信息。
- 图谱增量由 `BuildGraphUpdateFromEvent` 构建并合并推送（前端节流渲染）。

## 规则引擎与告警
- 规则来源：`config/enhanced_security_config.yaml`（优先）或回退到 `config/security_rules.yaml`。
- 支持类别优化、节流、告警历史、白名单；告警附带 MITRE 技术映射与攻击链检测（`event_context.go`）。

## API 概要
- WebSocket：`/ws`，推送 `stats`、`event`、`alert`、`graph_update`。
- REST（示例）：`/api/alerts`、`/api/stats` 等（由 `router.go` 注册）。
- 鉴权：当前已禁用；如需开启，通过中间件 `cors.go` 的 `RequireAuth` 控制并使用令牌。

## 数据与存储
- 可选 SQLite 存储（`storage_sqlite.go`）；默认内存结构。
- 数据目录 `data/` 用于持久化文件（可按需启用）。

## 性能与稳定性
- WS 写泵含 Ping/Pong 心跳；客户端队列默认 1024（可通过 `ETRACEE_WS_QUEUE_SIZE` 调整）。
- 前端对图谱更新节流 500ms，力导布局加入中心力与碰撞约束，避免持续放大与越界。
- 事件过滤：`pid/uid` 范围、类别开关；默认 UID 范围 `[0,65535]`。

## 运行与配置
- 绑定地址：`ETRACEE_BIND_ADDR`（默认 `0.0.0.0:8888`）。
- 令牌：`ETRACEE_API_TOKEN`（当前未启用）。
- 图谱与前端：无 D3 时功能降级但连接正常。

## 常见问题与排查
- WS 卡"连接中"或"已断开"：
  - 检查前端 WebSocket 路径是否为 `/ws`（前端 `index.html:436`）
  - 检查后端路由是否正确注册 `/ws`（`webserver/main.go:162`）
  - 检查认证中间件是否正确跳过 `/ws` 路径（`auth.go:428-436`）
  - 检查 CORS 中间件是否允许 WebSocket 升级
- UID/GID 恒为 1000：将 `MinUIDFilter` 设为 `0`；已默认修复。
- syscall_id 为 0：非系统调用入口事件（如退出）属预期；已增加更多 `sys_enter_*` 跟踪点降低比例。
