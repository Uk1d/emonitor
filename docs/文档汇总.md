# eTracee 文档汇总

## 项目概述

eTracee 是一个基于 eBPF 的轻量级安全监控与攻击链可视化系统，专为国产操作系统（如 openEuler）优化设计。

## 核心功能

- **eBPF 内核监控**：使用 CO-RE 技术实现跨内核版本兼容
- **实时威胁检测**：基于 Falco/Tracee 风格的规则引擎
- **AI 异常检测**：使用 Python 实现的统计方法检测异常行为
- **AI 应急响应助手**：支持多种 AI 服务商，提供实时安全分析
- **多格式报告导出**：支持 JSON/CSV/HTML 三种报告格式
- **攻击链可视化**：力导向图展示攻击路径

## 架构概览

### 三层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     Web 前端 (单页应用)                          │
│              告警列表 | 事件流 | 攻击链图谱 | AI 对话            │
└─────────────────────────────────────┬───────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────┐
│              Go 服务层 (规则引擎 + WebSocket + API)              │
│         事件处理 | 规则匹配 | 告警管理 | 攻击链检测              │
└─────────────────────────────────────┬───────────────────────────┘
                                      │
                    ┌─────────────────┴─────────────────┐
                    │                                   │
                    ▼                                   ▼
┌───────────────────────────────────┐   ┌───────────────────────────────────┐
│         eBPF 内核层               │   │        Python AI 服务             │
│    系统调用跟踪 | 事件采集        │   │    异常检测 | 报告生成           │
└───────────────────────────────────┘   └───────────────────────────────────┘
```

### 数据流

```
eBPF 程序 → Ring Buffer → Go 用户态 → 规则引擎 → 告警管理器
                ↓                              ↓
           事件解析                        攻击链检测
                ↓                              ↓
           WebSocket ←──────────────────── 广播推送
                ↓
           Web 前端 ←→ MySQL 存储
```

## 核心模块

### 1. eBPF 内核层 (`src/bpf/`)

| 文件 | 功能 |
|------|------|
| `etracee.h` | 事件结构定义 |
| `etracee_main.c` | eBPF 主程序 |
| `execve_trace.c` | 进程跟踪（execve, fork, exit） |
| `filesystem_trace.c` | 文件系统跟踪（open, read, write, chmod） |
| `network_trace.c` | 网络跟踪（connect, bind, listen, accept） |
| `security_trace.c` | 安全事件跟踪（ptrace, kill, mount） |

### 2. Go 用户态层 (`src/go/`)

| 文件 | 功能 |
|------|------|
| `main.go` | 主入口，eBPF 程序加载和事件处理循环 |
| `rule_engine.go` | 规则引擎，安全规则匹配 |
| `alert_manager.go` | 告警管理器，告警处理和聚合 |
| `alert_api.go` | Web API 和 WebSocket 服务 |
| `event_context.go` | 事件上下文，攻击链检测 |
| `storage_mysql.go` | MySQL 存储后端 |
| `internal/auth/` | 认证服务（JWT + MySQL） |
| `internal/web/static/` | Web 前端静态资源 |

### 3. Python AI 服务 (`src/python/`)

| 文件 | 功能 |
|------|------|
| `service.py` | Flask 服务入口 |
| `ai_detector/detector.py` | AI 异常检测器 |
| `report_generator/` | 报告生成器 |

## 运行模式

### 1. AI 增强模式（推荐）

```
终端 1: cd src/python && ./start.sh  # Python AI 服务 (端口 9900)
终端 2: sudo ./bin/etracee -monitor-only  # 监控程序 (需要 root)
终端 3: ./bin/webserver  # Web 服务 (端口 8888)
```

### 2. 分离模式

```
终端 1: sudo ./bin/etracee -monitor-only  # 监控程序 (需要 root)
终端 2: ./bin/webserver  # Web 服务 (端口 8888)
```

### 3. 集成模式

```
sudo ./bin/etracee  # 监控程序和 Web 服务在同一进程
```

## 数据库设计

| 数据库 | 用途 | 存储内容 |
|--------|------|----------|
| `etracee_events` | 监控程序数据库 | 事件、告警、攻击链 |
| `etracee_web` | Web 服务数据库 | 用户、会话、权限 |

## WebSocket 通信

- **监控程序 (端口 8889)**：推送事件和告警到独立 Web 服务
- **Web 服务 (端口 8888)**：接收数据并转发到浏览器

消息类型：
- `stats`：统计数据
- `event`：系统事件
- `alert`：安全告警
- `graph_update`：攻击链更新

## Web 界面功能

### 主要模块

1. **告警列表**：实时显示安全告警，支持按级别过滤
2. **事件流**：实时显示系统事件，支持搜索
3. **攻击链图谱**：可视化展示攻击路径
4. **AI 检测**：AI 异常检测结果（Beta）
5. **报告导出**：生成 JSON/CSV/HTML 格式报告
6. **AI 对话**：应急响应 AI 助手

### AI 对话功能

支持多种 AI 服务商：
- OpenAI (GPT-3.5/4/4o)
- Anthropic (Claude 3/3.5)
- Moonshot (Kimi)
- Google (Gemini)
- DeepSeek
- 智谱 AI (GLM-4)
- 阿里云 (通义千问)
- 百度 (文心一言)
- 硅基流动 (SiliconFlow)

## 常见问题

1. **BTF 不支持**：升级内核到 5.15+ 或手动生成 vmlinux.h
2. **权限不足**：监控程序需要 root 权限
3. **数据库连接失败**：检查 MySQL 服务和配置
4. **WebSocket 连接失败**：检查端口是否被占用
